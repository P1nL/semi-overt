# 页面手动回归脚本

适用版本：当前主分支前端实现（stores + features + widgets + pages）。

## A. 执行前准备
- 启动后端与前端
- 浏览器清理本地缓存（可选）
- 准备 `USER` 与 `ADMIN` 两类账号

## B. 核心回归路径（按顺序）

1. 认证基础
- 打开登录页，错误密码登录应提示失败
- 正常登录成功后，Header 显示用户态
- 退出登录后回到访客态

2. 首页与分类
- 首页可加载 Hero 与分类区块
- 点击分类菜单跳转分类页，列表正常显示

3. 搜索流程
- 在首页或搜索页输入关键词搜索
- URL query 与结果列表同步
- 空关键词时不应报错

4. 编辑器新建与保存
- 从“新建文章”进入编辑页
- 输入标题/摘要/正文，触发自动保存
- 首次保存后应自动切到 `/editor/:id`
- 刷新页面后内容仍可从接口拉回

5. 草稿箱
- Header 打开草稿箱，看到刚保存草稿
- 点击“继续编辑”应跳到对应编辑页
- 点击“删除”后草稿消失且角标更新

6. 发布动作
- 在 `DRAFT/RETURNED` 状态可提交审核
- 提交后状态应变为 `PENDING`
- 在 `PENDING` 可执行“取消审核”并回到 `DRAFT`
- 在不可操作状态按钮应禁用或提示

7. 审核工作台（管理员）
- 管理员登录进入审核页与审核详情页
- 对同一文章执行通过/退回/拒绝
- 退回与拒绝必须填写原因
- 操作后日志区可看到新增记录

8. 个人主页与资料编辑
- 打开个人主页，资料与文章列表显示正确
- 编辑资料（头像/封面/签名）保存成功
- Header 和个人主页显示同步更新

9. 全局交互
- 切换主题后刷新，主题状态保持
- Toast 在成功/失败操作后可见
- 404 页面与 403 页面可正常显示

10. 权限与异常
- 手动构造无 token 请求，页面应走 401 处理链路
- 非管理员访问审核接口应走 403 处理链路
- 跳转不存在路由应进入 404 页面

## C. 验收标准
- 不出现白屏、致命报错、无限加载
- 核心业务路径（登录 -> 编辑 -> 提交 -> 审核 -> 查看）全通
- 状态流转和按钮可用性与文章状态一致
- 401/403/404 行为与预期一致
