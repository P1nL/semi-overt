## Design Context

### Users
Primary users are Chinese-speaking readers browsing curated long-form and short-form articles, plus signed-in authors and reviewers moving content through draft, submission, and review workflows. Public visitors need to understand what is worth reading quickly; authenticated users need low-friction paths into writing, review status, and profile management.

### Brand Personality
Calm, editorial, trustworthy. The interface should feel considered rather than promotional, premium without feeling luxurious, and confident without sounding corporate.

### Aesthetic Direction
Light-first, soft-surface, content-led UI with restrained blue accents and subtle motion. The product already leans toward Apple-adjacent frosted chrome and minimal typography; future work should keep that restraint while avoiding generic AI-SaaS tropes, decorative gradients, and dashboard-card excess.

### Design Principles
1. Content first: every public screen should help users decide what to read next within a few seconds.
2. One dominant action: each screen should make the primary next step obvious without forcing users to inspect multiple equal-weight elements.
3. Calm over clever: motion, blur, and visual texture should support orientation, not become the main event.
4. Reduce decision cost: labels, summaries, and status copy should explain value and state in plain language.
5. Human review flows: authoring, submission, and review states should feel legible and reassuring, especially around high-stakes actions.
